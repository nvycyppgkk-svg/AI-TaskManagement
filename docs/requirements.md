# 要件定義書

## プロジェクト概要

| 項目 | 内容 |
|---|---|
| プロジェクト名 | Trello風タスク管理アプリ |
| 作成日 | 2026-05-07 |
| バージョン | 1.0 |

個人向けTrello風タスク管理Webアプリ。PC・スマートフォンのブラウザで動作する。

---

## 背景・目的

ITスクールのカリキュラム課題として、Webアプリケーション開発の一連の工程（要件定義→設計→実装→テスト）を体験することを目的とする。

題材として、実際に自分が使えるタスク管理アプリを開発する。

---

## ユーザー

- 1人（ログイン不要）

---

## システム構成

フロントエンド・バックエンド・DBの3層構成とする。

```mermaid
graph TD
  A[ブラウザ（フロントエンド）] -->|API通信| B[バックエンドサーバー]
  B -->|読み書き| C[データベース（DB）]
```

---

## ユースケース

| # | アクター | ユースケース |
|---|---|---|
| UC-1 | ユーザー | ボードを作成・切り替え・削除する |
| UC-2 | ユーザー | ボード内にリストを追加・削除・名前変更する |
| UC-3 | ユーザー | リストにカードを追加し、タイトル・説明・期限・優先度を設定する |
| UC-4 | ユーザー | カードにラベルを作成して付与する |
| UC-5 | ユーザー | カードをドラッグ&ドロップでリスト間・リスト内で移動する |
| UC-6 | ユーザー | カードの詳細をモーダルで確認・編集する |
| UC-7 | ユーザー | カードを削除する |

---

## 操作フロー

### メインフロー：カードを作成して管理する

```mermaid
flowchart TD
  A[アプリを開く] --> B[ボードを選択または新規作成]
  B --> C[リストを選択]
  C --> D[「カードを追加」をクリック]
  D --> E[タイトルを入力して保存]
  E --> F[カード詳細モーダルを開く]
  F --> G[説明・期限・優先度・ラベルを設定して保存]
  G --> H[必要に応じてドラッグ&ドロップで移動]
```

### サブフロー：ボードを新規作成する

```mermaid
flowchart TD
  A[「新しいボードを作成」をクリック] --> B[ボード名を入力して確定]
  B --> C[「To Do / 進行中 / 完了」の3リストが自動生成]
  C --> D[ボード詳細画面が表示される]
```

### サブフロー：ラベルを作成してカードに付与する

```mermaid
flowchart TD
  A[カード詳細モーダルを開く] --> B[「ラベルを追加」をクリック]
  B --> C[名前・色を設定して保存]
  C --> D[カードにラベルが付与される]
```
